# novTo-Do List — Полная документация

## 📱 О проекте

**novTo-Do List** — Android-приложение для управления задачами в кибер-неоновом стиле.  
Разработчик: XenorKing | Nova Project  
Стек: Kotlin + Jetpack Compose + Firebase (Auth + Firestore) + WorkManager

## 🗄️ База данных (Firebase Firestore)

### Структура коллекций:

```
users/
  {uid}/                        ← документ пользователя
    displayName: String         ← никнейм
    nickname: String            ← никнейм (дубль для поиска)
    email: String
    fcmToken: String            ← токен для push-уведомлений
    
    tasks/                      ← подколлекция задач
      {taskId}/
        id: String
        title: String           ← название (обязательно)
        description: String     ← описание
        isCompleted: Boolean    ← выполнена ли задача
        completedBy: String?    ← uid пользователя, выполнившего
        completedByName: String? ← никнейм выполнившего
        dueDate: Timestamp?     ← дата и время дедлайна
        createdAt: Timestamp    ← дата создания
        ownerId: String         ← uid владельца
        priority: Int           ← 0=низкий, 1=средний, 2=высокий
        category: String        ← work/personal/health/study/other
        reminderOffset: Int     ← минут до уведомления (0/5/15/30/60/1440)
        repeatType: String      ← none/daily/weekly/monthly
```

### Правила безопасности (appfirestore.rules):
- Каждый пользователь видит ТОЛЬКО свои задачи
- Операции разрешены только для авторизованных пользователей
- `userId == request.auth.uid`

## 🔄 Синхронизация

Приложение использует **Firestore Realtime Listeners** (`addSnapshotListener`):

1. При открытии экрана подписываемся на коллекцию задач
2. Firestore мгновенно push'ит изменения на все подключённые устройства
3. Offline-режим: Firestore кэширует данные локально
4. При восстановлении сети — автосинхронизация

```kotlin
// Пример в TaskRepository
.addSnapshotListener { snap, e ->
    trySend(snap?.toObjects(Task::class.java) ?: emptyList())
}
```

**Результат**: Если вы создаёте задачу на телефоне, она мгновенно появится на планшете без перезапуска приложения.

## 🔔 Уведомления (Fix #4)

Система двойных уведомлений через **WorkManager**:

1. **Заблаговременное** — за N минут до дедлайна (настраивается: 0/5/15/30/60/1440 мин)
2. **Точное** — ровно в момент дедлайна

```kotlin
// WorkManager планирует уведомление через delayMs миллисекунд
OneTimeWorkRequestBuilder<TaskReminderWorker>()
    .setInitialDelay(delayMs, TimeUnit.MILLISECONDS)
    .build()
```

> ⚠️ WorkManager не гарантирует миллисекундную точность — отклонение до 15 минут.
> Для точных уведомлений можно добавить AlarmManager (требует разрешения SCHEDULE_EXACT_ALARM).

## 🐛 Известные проблемы и как их избежать

1. **WorkManager может опаздывать** с уведомлениями на Doze Mode устройствах.
   - Решение: попросить пользователя добавить приложение в исключения оптимизации батареи.

2. **displayName в Firebase Auth** обновляется не мгновенно.
   - При смене ника используем `updateProfile()` + обновляем в Firestore.

3. **Повторяющиеся задачи** (`repeatType != "none"`) — логика повторения ещё не реализована.
   - После выполнения повторяющейся задачи нужно создавать новую с сдвинутой датой.

4. **FCM-уведомления** (push через сервер) требуют бэкенда.
   - Текущая реализация: только локальные уведомления через WorkManager.
   - Для серверных push-уведомлений нужен Firebase Cloud Functions.

## 🔐 Безопасность

- Все запросы к Firestore проходят через Firebase Security Rules
- Пользователь видит только задачи с `ownerId == auth.uid`
- Пароли хранятся в Firebase Auth (bcrypt), приложение их не видит
- FCM-токены хранятся в Firestore под UID пользователя
